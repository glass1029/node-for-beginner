1.1 노드의 정의
- 자바스크립트 런타임(실행기)
- HTML과 브라우저로부터 종속성이 해소됨
- 내부는 c++로 구성됨 (처음부터 c++로 작성하면 생산성이 떨어지기에 등장)
- 비동기 구현해주는 libuv가 포함됨
- 싱글스레드 + 비동기

1.2 노드의 특성
- 이벤트 기반: 이벤트가 발생할 떄 미리 지정해둔 작업을 수행하는 방식
- 논블로킹: 노드는 블로킹이면서 동기, 비동기면서 논블로킹 (동기는 순서대로, 비동기는 순서대로 실행되지 않을 수도 있다)
- 프로세스: 운영체제에서 할당하는 작업의 단위 (프로세스 간 자원 공유x)
- 스레드: 프로세스 내에서 실행되는 작업의 단위 (부모 프로세스 자원 공유)
- 노드는 멀티 스레드이지만, 직접 다룰 수 있는 스레드는 하나이기 때문에 싱글 스레드라고 표현한다. (14버전부터는 멀티 스레드 사용 가능하지만, 코딩이 매우 어려움)
- 싱글 스레드는 에러를 처리하지 못하는 경우 멈춘다.
- 노드는 대신 논블로킹 모델을 채택하여 일부 코드를 백그라운드에서 실행 가능하다.

1.3 노드의 역할
- 노드는 서버가 아니지만 서버를 구성할 수 있게 하는 모듈을 제공한다.
- 멀티 스레드 방식에 비해서 컴퓨터 자원을 적게 사용하지만, 싱글스레드라서 CPU코어를 하나만 사용한다.
- I/O 작업이 많은 서버로 적합하지만, CPU작업이 많은 서버로는 부적합하다.
- 멀티 스레드 방식보다 쉽지만, 하나뿐인 스레드가 멈추지 않도록 관리해야한다.
- 웹서버가 내장되어있다.
- 서버 규모가 커졌을 때 서버를 관리하기 어렵다.
- json형식과 호환하기 좋다.

2.1 호출스택, 이벤트루프
- 자바스크립트가 순서대로 코드를 읽으면서 호출스택에 쌓인다.
- 이 때 setTimeout과 같은 비동기함수가 있다면 
  호출스택에서 setTimeout() 호출 -> 백그라운드로 보냄 -> 호출스택에서 제거 -> 
  지정 시간 뒤에 테스크큐로 실행예정 함수 보냄 -> 백그라운드에서 제거 ->
  호출스택이 비어있을 때 이벤트루프가 테스크큐의 함수들을 호출스택으로 보냄 ->
  호출스택에서 실행예정 함수 실행 -> 함수 종료되면 호출스택에서 제거 -> 자바스크립트 종료
- 호출스택과 백그라운드는 동시에 실행된다. (호출스택은 자바스크립트, 백그라운드는 c++)
- promise는 내부에서 동기로 실행되지만 .then()을 만나는 순간 비동기가 된다.
- 백그라운드에 여러 함수가 쌓이면 promise가 우선순위를 가진다.

2.2 ES 2015+
- this를 써야하면 function, 그 외는 모두 화살표 함수로 쓰는 것을 추천한다.
- function에서 this사용하는 경우는 구조분해할당 비추천한다.
- async는 항상 promise를 반환하므로 .then()이나 await을 붙여서 사용한다.
- async await 사용할 때는 try catch로 reject 캐치해야한다.
- for await of는 프로미스를 반복하는 loop에서 사용한다.
